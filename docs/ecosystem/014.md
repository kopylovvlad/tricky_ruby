# How to run prometheus exporter

It's easy to run prometheus exporter in web-application, but what about background jobs? For example, for delayed_jobs?

Solution: to run prometheus exporter manualy with webrick. Example:

```ruby
require 'webrick'
Yabeda::Prometheus::Mmap::Exporter.start_metrics_server!
```

Then, you can parse metrics by `curl -is localhost:9394/metrics`

## Another solution with port cheking

```ruby
module Metrics
  PROMETHEUS_PORT = '9394'
  PROMETHEUS_BIND = '0.0.0.0'

  class < self
    def start_metrics_server!
      return unless port_available?

      Yabeda::Prometheus::Exporter.start_metrics_server!(port: PROMETHEUS_PORT, bind: PROMETHEUS_BIND)
    rescue StandardError => e
      Rails.logger.error "Failed to start metrics server: #{e message}."
    end

    def port_available?
      Socket.tcp_server_sockets(PROMETHEUS_BIND, PROMETHEUS_PORT).each(&:close)
      true
    rescue Errno::EADDRINUSE => e
      Rails.logger.error "Failed to start metrics server: #{e.message}"
      false
    end
  end
end

Metrics.start_metrics_server!
```
